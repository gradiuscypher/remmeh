**`remmeh/ui/widgets/status_bar.py`:**
- Subclass `Static` (not Widget — Static already handles rendering)
- Renders: `f"[bold $accent]Model:[/] {model_name}  [dim]remmeh[/]"` — model left, app name right
- `update_model(model)` calls `self.update(...)` with new text
- `set_streaming(True)` prepends `"⠋ Streaming…  "` spinner text; `set_streaming(False)` removes it
- Store model_name and streaming state as instance vars; `_render()` helper builds the content string

**`remmeh/ui/widgets/input_panel.py`:**
- Subclass `Widget`
- `compose()` mounts a `TextArea` with `id="input-area"`, `tab_behavior="indent"` set to False
- Configure TextArea: no line numbers, `soft_wrap=True`, placeholder text per UI spec
- **Enter handling:** Override `on_key(event: Key)` — if `event.key == "enter"` and NOT `event.is_printable`, consume the event, get TextArea value, post `InputPanel.Submitted(text)`, clear TextArea
  - `event.prevent_default()` to stop Enter from inserting newline
  - Only fire if text is not empty after stripping whitespace
- **Shift+Enter:** Let it pass through to TextArea (inserts newline — default behavior)
- **CRITICAL:** Do NOT use `BINDINGS = [("enter", ...)]` at app/screen level — that conflicts with TextArea editing. Use `on_key` scoped to InputPanel only (per UI spec pitfall #9).
- `disable()`: sets TextArea `read_only = True`, updates border color
- `enable()`: sets TextArea `read_only = False`

**`remmeh/ui/widgets/chat_view.py`:**
- Subclass `VerticalScroll` (NOT just Widget — VerticalScroll handles scrolling built-in)
- `compose()`: renders empty state when no messages — a `Static("Start a conversation", classes="empty-heading")` and `Static("Type a message below to begin chatting.", classes="empty-body")`
- `add_message_widget(widget)`: if empty state showing, remove it; `self.mount(widget)`; then `self.scroll_end(animate=False)`
- **CRITICAL:** Do NOT use `height: auto` — use `height: 1fr` in TCSS (per pitfall #8)
- `scroll_to_end()`: `self.scroll_end(animate=False)`
